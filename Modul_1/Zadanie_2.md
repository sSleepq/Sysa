## 1. Адресация на интерфейсах

### 1.1. Конфигурация

Файл: `/etc/network/interfaces`

```
auto eth0
iface eth0 inet dhcp

auto eth1
iface eth1 inet static
    address 172.16.1.1/28

auto eth2
iface eth2 inet static
    address 172.16.2.1/28
```

### 1.2. Назначение интерфейсов

| Интерфейс | Назначение              | Тип адресации |
|-----------|-------------------------|---------------|
| eth0      | В сторону Интернета     | DHCP          |
| eth1      | В сторону офиса HQ      | Статическая   |
| eth2      | В сторону офиса BR      | Статическая   |

### 1.3. Применение

```
systemctl restart networking
```

## 2. Включение маршрутизации

Для пересылки пакетов между интерфейсами необходимо активировать IP-форвардинг.

### 2.1. Редактирование sysctl

Файл: `/etc/sysctl.conf`

Добавлена строка:

```
net.ipv4.ip_forward=1
```

### 2.2. Применение

```
sysctl -p
```

## 3. Настройка NAT

Настройка трансляции сетевых адресов для обеспечения доступа локальных сетей в Интернет.

### 3.1. Добавление правила

```
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
```

Пояснение:

| Параметр          | Значение                                           |
|-------------------|----------------------------------------------------|
| `-t nat`          | Работа с таблицей NAT                              |
| `-A POSTROUTING`  | Добавление правила в цепочку POSTROUTING           |
| `-o eth0`         | Применение к пакетам, исходящим через интерфейс eth0 |
| `-j MASQUERADE`   | Подмена源ного IP-адреса на IP-адрес интерфейса eth0  |

---

## 4. Автозагрузка правил

Обеспечение автоматического применения настроек после перезагрузки устройства.

### 4.1. Сохранение правил iptables

```
iptables-save > /etc/rules.v4
```

### 4.2. Добавление задач в crontab

Команда для редактирования:

```
crontab -e
```

Добавлены строки:

```
@reboot /sbin/iptables-restore < /etc/rules.v4
@reboot /sbin/sysctl -p
```

| Строка                                                 | Назначение                                         |
|--------------------------------------------------------|----------------------------------------------------|
| `@reboot /sbin/iptables-restore < /etc/rules.v4`       | Восстановление правил iptables при загрузке        |
| `@reboot /sbin/sysctl -p`                              | Применение параметров sysctl при загрузке          |
