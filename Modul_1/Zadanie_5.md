## 1. Требования

| Параметр                         | Значение                       |
|----------------------------------|--------------------------------|
| Порт подключения                 | `2026`                         |
| Разрешённый пользователь         | `sshuser`                      |
| Максимальное количество попыток  | `2`                            |
| Баннер                           | `Authorized access only`       |

## 2. Настройка на HQ-SRV

**Устройство:** HQ-SRV

### 2.1. Редактирование конфигурации SSH

Файл: `/etc/openssh/sshd_config`

Команда для редактирования:

```
mcedit /etc/openssh/sshd_config
```

Внесены следующие параметры:

| Параметр       | Значение               | Пояснение                                      |
|----------------|------------------------|------------------------------------------------|
| `Port`         | `2026`                 | Смена стандартного порта SSH                   |
| `AllowUsers`   | `sshuser`              | Разрешение подключения только пользователю sshuser |
| `MaxAuthTries` | `2`                    | Ограничение количества попыток аутентификации  |
| `Banner`       | `/root/banner`         | Путь к файлу баннера                           |

Файл сохраняется, редактор закрывается.

### 2.2. Создание баннера

**Устройство:** HQ-SRV

Файл: `/root/banner`

Команда для создания:

```
mcedit /root/banner
```

Содержимое:

```
Authorized access only

```

Примечание: после строки баннера обязательно оставить пустую строку.

Файл сохраняется, редактор закрывается.

### 2.3. Применение и проверка

**Устройство:** HQ-SRV

Перезапуск службы:

```
systemctl restart sshd
```

Добавление в автозагрузку:

```
systemctl enable sshd
```

**Проверка 1.** Подключение под пользователем, отличным от `sshuser`.

```
ssh user@localhost -p 2026
```

Результат:

```
Authorized access only
user@localhost's password:
Permission denied, please try again.
```

Подключение отклонено.

**Проверка 2.** Подключение под пользователем `sshuser`.

```
ssh sshuser@localhost -p 2026
```

Результат:

```
Authorized access only
sshuser@localhost's password:
[sshuser@hg-sru ~]$
```

Подключение успешно.

## 3. Настройка на BR-SRV

**Устройство:** BR-SRV

Действия, описанные в разделах 2.1–2.3, полностью повторяются на сервере **BR-SRV**:

1. Редактирование файла `/etc/openssh/sshd_config` с теми же параметрами.
2. Создание файла `/root/banner` с содержимым `Authorized access only`.
3. Перезапуск службы `sshd` и добавление в автозагрузку.
4. Проверка подключения под пользователями `user` и `sshuser`.
